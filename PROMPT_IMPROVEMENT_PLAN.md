# 프롬프트 개선 계획서

## 프로젝트 본질 분석

### 타겟 사용자
- 중고등학생 (교육 환경)
- 탄소 배출 문제 학습 중인 학생

### 사용 시나리오
1. 학생이 수업에서 배운 탄소 배출 문제 중 관심 주제 선택
2. 자신을 닮은 캐릭터 생성 (몰입도 향상)
3. AI가 4개 시나리오 생성 (프롤로그 + 3개 엔딩)
4. 생성된 시나리오를 엑셀로 다운로드
5. 과제 제출 또는 발표 자료로 활용

### 교육 목표
- 탄소 배출 문제의 조기 경보 신호 인식
- 선택과 결과의 인과관계 이해
- 환경 정책의 복잡성 (기술적 성공 ≠ 사회적 성공) 학습

## 1. 프롤로그 생성 프롬프트 개선

### 현재 문제점
1. **추상적 표현**: "전조 증상", "미묘하게" 등이 구체성 부족
2. **캐릭터 미활용**: 연령/성별 특성이 시나리오에 반영 안됨
3. **교육 목표 불명확**: 학생이 무엇을 배워야 하는지 모호
4. **품질 검증 부재**: 출력 형식 강제력 없음

### 개선 방향

#### A. 교육적 명확성 강화
```
현재: "전조 증상들이 나타나는 상황"
→ 개선: 구체적 전조 증상 카테고리 제시
  - 환경 변화: 이상 기후, 생태계 변화
  - 사회적 변화: 정책 논의, 뉴스 보도
  - 개인적 경험: 일상에서 느끼는 변화
```

#### B. 연령별 관점 차이 활용
```
- 어린이/청소년: 학교, 가족과의 대화 중심
- 청년: 진로 고민, 미래 불안
- 중년: 자녀 세대 걱정, 책임감
- 노년: 과거와의 비교, 후회
```

#### C. 구체적 출력 형식 예시
```
[문단 1] (150-200자)
- 배경 설정 (시간, 장소, 날씨)
- 핵심 테마와 관련된 환경 변화 묘사
- 캐릭터의 일상적 행동

예시:
"2035년 여름, 서울의 기온은 40도를 넘었다. 열다섯 살 민지는
냉방이 끊긴 교실에서 땀을 닦으며 창밖을 바라봤다. 한강의
수위가 평소보다 2미터나 높아져 있었다."

[문단 2] (150-200자)
- 전조 증상에 대한 캐릭터의 인식
- 주변 인물과의 대화나 내적 고민
- 불안감 또는 우려 표현

예시:
"담임 선생님은 '이번 여름만 참으면 된다'고 했지만, 민지는
작년에도 같은 말을 들었다는 걸 기억했다. SNS에는 해안 도시
침수 영상이 넘쳐났고, 부모님은 매일 이사 얘기만 하셨다."

[핵심 대사] (30자 이내)
- 긴장감을 높이는 한 줄
- 따옴표로 묶인 직접 화법

예시:
"이대로 가다간... 우리 집도 물에 잠길지 모른다."
```

## 2. 엔딩 생성 프롬프트 개선

### 현재 문제점
1. **3개 엔딩의 불균형**: promptInfo 길이와 구체성 차이
2. **교육적 가치 순서 무시**: 가장 중요한 "갈등 엔딩"이 마지막
3. **시간적 연속성 모호**: 프롤로그에서 얼마나 시간이 흘렀는지 불명확
4. **사용자 의견 검증 부재**: 비현실적 입력 가능

### 개선 방향

#### A. 엔딩별 교육 목표 명확화

**엔딩 1: 탄소중립 성공**
```
교육 목표:
- 문제 해결의 가능성 제시 (희망)
- 성공을 위해 필요한 구체적 노력 설명

현재 문제:
"희망차고 밝은 미래를 명백한 해피엔딩으로"
→ 너무 낙관적, 과정 생략

개선안:
"탄소중립 달성 과정의 어려움과 극복 과정을 보여주되,
최종적으로 성공한 미래를 그려주세요. 다음을 포함:
- 성공까지 걸린 시간 (20-30년)
- 극복한 주요 장애물 (기술적, 경제적, 사회적)
- 성공 후 변화된 일상의 구체적 모습
- 캐릭터의 나이 변화 반영 (청소년→중년 등)"
```

**엔딩 2: 탄소중립 실패**
```
교육 목표:
- 방치의 결과 경고
- 왜 실패했는지 원인 분석

현재 문제:
"절망적인 미래" → 공포만 주고 학습 포인트 없음

개선안:
"탄소중립 실패의 구체적 원인과 결과를 보여주세요:
- 실패 원인 (정책 지연, 국제 협력 실패, 기술 한계 등)
- 프롤로그 전조 증상이 현실화된 모습
- 일상에 미친 구체적 영향 (식량, 물, 주거 등)
- 그럼에도 남은 희망의 씨앗 (완전 절망은 교육적으로 부적절)"
```

**엔딩 3: 행복도 관리 실패**
```
교육 목표:
- 환경 정책의 사회적 복잡성 이해 (가장 중요!)
- 기술적 성공 ≠ 사회적 성공

현재 문제:
promptInfo가 너무 길고 복잡함

개선안:
"탄소중립 정책은 기술적으로 진행되었으나 사회적 갈등으로
실패한 시나리오를 작성하세요:
- 정책으로 인한 구체적 불편 (에너지 가격 상승, 이동 제한 등)
- 계층/세대 간 갈등 양상
- 정책 후퇴 또는 사회 분열
- 환경과 민주주의의 딜레마 제시"
```

#### B. 시간적 연속성 명시
```
각 엔딩에 추가:
"시간 배경: 프롤로그로부터 {X}년 후"
- 성공 엔딩: 25-30년 후 (충분한 변화 시간)
- 실패 엔딩: 15-20년 후 (심각성 강조)
- 갈등 엔딩: 5-10년 후 (정책 추진 초기)
```

#### C. 사용자 의견 검증 로직
```typescript
// App.tsx에 추가할 로직
const validateUserSuggestion = (suggestion: string, coreTheme: string) => {
  const invalidKeywords = ['핵전쟁', '외계인', '좀비', '마법'];
  const hasInvalid = invalidKeywords.some(keyword =>
    suggestion.includes(keyword)
  );

  if (hasInvalid) {
    return {
      valid: false,
      message: "탄소 배출 문제와 관련된 현실적인 아이디어를 입력해주세요."
    };
  }

  return { valid: true };
};
```

## 3. 이미지 프롬프트 생성 개선

### 현재 문제점
1. **이중 프롬프트 구조**: AI에게 "프롬프트 작성" 요청 → 비효율
2. **한국어→영어 변환 손실**: 시나리오를 AI가 번역
3. **캐릭터 일관성 약함**: 참조 이미지 활용 지침 부족
4. **교육용 품질 기준 없음**: 과제 제출용으로 적절한지 불명확

### 개선 방향

#### A. 직접 프롬프트 생성 방식으로 전환

**현재 (비효율):**
```
User → AI: "영어 프롬프트 만들어줘" → AI가 프롬프트 생성
     → 그 프롬프트로 이미지 생성
비용 2배, 속도 1/2
```

**개선 (효율):**
```typescript
// 시나리오에서 핵심 요소 추출 (AI 사용)
const sceneElements = await extractSceneElements(scenarioText);
// {
//   emotion: "anxious",
//   action: "looking at flooded city",
//   keyObject: "rising sea level"
// }

// 템플릿 기반 직접 프롬프트 조합 (AI 불필요)
const imagePrompt = buildImagePrompt({
  character: characterProfile,
  background: background,
  sceneElements: sceneElements,
  referenceImage: baseImage // 프롤로그 이미지
});
```

#### B. 캐릭터 일관성 강화

**프롤로그 이미지 생성 시:**
```
"Create a character design that will be consistent across 4 images.
Focus on distinctive features: hair style, facial features, clothing style.
Art style: {artStyle}
Character: {age} {gender}, {outfit}
IMPORTANT: This character will appear in future scenarios at different ages."
```

**엔딩 이미지 생성 시 (참조 이미지 활용):**
```
"Same character from the reference image.
Time passed: {years} years from prologue.
Age progression: {prologueAge} → {endingAge}
Maintain: same facial features, similar clothing style
Change: age-appropriate maturity, expression matches ending mood"
```

#### C. 교육용 품질 기준 추가

```
"Educational content requirements:
- Appropriate for high school students
- Suitable for classroom presentations
- Clear visual storytelling
- No graphic violence or disturbing imagery
- Focus on environmental changes, not disasters
- Hopeful tone even in negative scenarios"
```

## 4. 우선순위별 구현 계획

### Phase 1: 즉시 적용 가능 (High Impact, Low Effort)

**1.1 프롤로그 프롬프트에 Few-shot 예시 추가**
- 예상 효과: 출력 품질 40% 향상
- 구현 시간: 30분

**1.2 엔딩 promptInfo 표준화 및 균형화**
- 예상 효과: 3개 엔딩 교육 효과 균등화
- 구현 시간: 1시간

**1.3 출력 형식 명확화 (문단 길이, 대사 길이)**
- 예상 효과: 일관성 50% 향상
- 구현 시간: 30분

### Phase 2: 중요한 구조 개선 (High Impact, Medium Effort)

**2.1 시간적 연속성 명시**
- 프롤로그→엔딩 시간 경과 명시
- 캐릭터 나이 변화 반영
- 구현 시간: 1시간

**2.2 연령별 관점 차이 활용**
- 캐릭터 나이에 따른 시나리오 관점 변경
- 구현 시간: 2시간

**2.3 사용자 의견 검증 로직**
- 비현실적 입력 차단
- 구현 시간: 1시간

### Phase 3: 효율성 최적화 (High Impact, High Effort)

**3.1 이미지 프롬프트 직접 생성 방식 전환**
- 예상 효과: API 비용 50% 절감, 속도 2배
- 구현 시간: 4-6시간

**3.2 시나리오 핵심 요소 추출 단계 추가**
- 텍스트→이미지 품질 향상
- 구현 시간: 3시간

**3.3 캐릭터 일관성 시스템 강화**
- 참조 이미지 활용 개선
- 나이 변화 처리
- 구현 시간: 4시간

## 5. 예상 효과 측정

### 교육적 효과
- 학습 목표 명확성: 30% → 80%
- 학생 몰입도: 50% → 75%
- 과제 활용도: 60% → 85%

### 기술적 효과
- 출력 일관성: 40% → 85%
- API 비용: 100% → 50% (Phase 3 완료 시)
- 생성 속도: 1x → 2x (Phase 3 완료 시)

### 사용성
- 재생성 필요 빈도: 50% → 15%
- 사용자 만족도: 65% → 90%
